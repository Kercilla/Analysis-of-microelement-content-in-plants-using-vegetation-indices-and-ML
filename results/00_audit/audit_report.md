# Аудит пайплайна на data leakage

## Что проверялось

| Проверка | Описание | Что значит плохой результат |
|---|---|---|
| Baseline (среднее) | R² предсказания среднего | > 0.01 — баг в формуле R² |
| Ridge (5 каналов) | Физический минимум сигнала | Превышение GBR > +0.5 — подозрительно |
| Random KFold vs Spatial CV | Разница = оптимистическое смещение | Δ > 0.2 — пространственная автокорреляция |
| Permutation test | p < 0.05 — модель значима | p ≥ 0.05 — результат случайный |
| MixUp before/after split | Разница в R² | Δ > 0.05 — MixUp до split раздувает |

## Вывод по ml_pipeline.py

**StandardScaler**: применяется ВНУТРИ CV-fold (строки 99-101 ml_pipeline.py) — ✓ КОРРЕКТНО

**MixUp в 07_ml_enhanced.py**: применяется ко ВСЕЙ выборке ДО передачи в compare_models.
Это означает что синтетические MixUp-образцы могут попасть и в train и в test.
compare_models внутри делает KFold — но делает его по АУГМЕНТИРОВАННЫМ данным.

**Итог по MixUp**: это потенциальный leakage. Правильный вариант — MixUp только внутри train-fold.
